## Getting Started
1. Clone the repository
2. Create feature branches from `dev`
3. Submit PRs to `dev` for review
4. Merges to `main` trigger production deployment

## Branch Strategy
- `main` - Production (protected)
- `stage` - Staging environment (protected)
- `dev` - Development (protected)
- `feature/*` - Feature branches

## Required Reviews
All PRs require:

- Passing CI checks
- Work item linked
